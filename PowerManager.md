# PowerManager — Архитектура управления мощностью нагрева

## Обзор

Текущая схема управления ТЭНом: внешний регулятор с предустановленными значениями, ESP32 через контактор включает/выключает ТЭН и двумя пинами задаёт режим (полная мощность / предустановленная). Цель — заменить внешнюю схему на встроенное плавное управление мощностью напрямую с ESP32.

## Предлагаемая связка: ZMPT101B + DIMMERLINK (UART)

### Компоненты

**1. DIMMERLINK (UART) — RobotDyn или аналог**
- AC dimmer модуль с детектором нуля (zero-cross) и интерфейсом UART
- Управляет TRIAC, задавая угол открытия полуволны
- ESP32 отправляет команду → модуль сам обрабатывает zero-cross и управляет TRIAC
- Принимает уровень 0-10000 (шаг 0.01%)
- Ток: не менее 25А, рекомендуется 40А с запасом (ТЭН 3500Вт ≈ 16А при 220В)
- Обязательно радиатор — TRIAC греется
- Библиотека DIMMERLINK ~100 строк кода

**2. ZMPT101B — модуль измерения напряжения сети**
- Маленький трансформатор напряжения, гальванически развязан от сети
- Выход: переменное напряжение ~0-5В (пропорционально 0-250В сети)
- Подключение к аналоговому входу ADC ESP32
- Нужен делитель напряжения для ограничения до 3.3В (макс. на ADC GPIO ESP32)
- RC-фильтр для сглаживания пульсаций
- Цена: ~$1-2

### Схема подключения

```
Сеть 220В
    │
    ├──── ZMPT101B ──── RC-фильтр ──── Делитель ────→ ADC GPIO ESP32 (меряем напряжение)
    │
    ├──── DIMMERLINK ←──── UART TX/RX ESP32 (задаём угол)
    │         │
    │        ТЭН (нагрев)
    │
    └──────────────────────────────────────────────────────────
```

**Пины ESP32:**
- UART TX → DIMMERLINK RX
- UART RX → DIMMERLINK TX (опционально, для чтения состояния)
- 1 GPIO ADC → ZMPT101B через делитель и фильтр
- GND — общая земля с модулями

## Алгоритм стабилизации мощности

### Принцип

DIMMERLINK стабилизирует **угол открытия** (phase angle), но не **мощность**. При колебаниях напряжения сети (220В ± 15В) реальная мощность на ТЭНе меняется. ZMPT101B обеспечивает обратную связь — ESP32 мерит напряжение и корректирует угол.

### Формулы

```
R_TEN    — сопротивление ТЭНа (Ом), из настроек
U        — текущее напряжение сети (В), с ZMPT101B
P_target — заданная мощность (Вт)

Максимальная мощность при 100% угле:
  P_max = U² / R_TEN

Доля мощности:
  fraction = P_target / P_max

Угол открытия (линеаризация sin²):
  angle = arcsin(√fraction) × (10000 / (π/2))

Отправка на DIMMERLINK:
  dimmer.setLevel(angle)   // 0-10000
```

### Цикл стабилизации (каждый loop ~150 раз/сек)

```
1. readVoltage()       — среднее из 50-100 замеров ADC за полпериода (10 мс при 50 Гц)
2. calcCurrentPower()  — P_actual = U² × k(angle) / R_TEN
3. adjustAngle()       — PI-регулятор, если P_actual ≠ P_target
4. dimmer.setLevel()   — новая команда на диммер
```

### PI-регулятор

```
error = P_target - P_actual
integral += error × dt
output = Kp × error + Ki × integral
new_angle = constrain(current_angle + output, 0, 10000)
```

Коэффициенты Kp и Ki подбираются экспериментально. ТЭН инерционный — реакция на изменение мощности занимает 10-30 секунд, поэтому PI-регулятор не должен быть агрессивным.

## Класс PowerManager

```

│   ├── setLevel(0-10000)

│
├── voltageSensor (ADC, ZMPT101B)

│

```

Примерный объём: 200-300 строк C++.

## Проблемы и решения

### 1. ADC ESP32 нелинейный и шумный

ESP32 ADC имеет нелинейность и шум ~20-30 мВ. Для ZMPT101B это погрешность ~1-2В на 220В — приемлемо для данной задачи.

**Решения:**
- Среднее из 50-100 замеров за полпериода (10 мс при 50 Гц)
- Калибровка: замерить ADC при известном напряжении, вычислить коэффициент
- Альтернатива: внешний ADC ADS1115 (I2C, 16-бит) — но это оверкилл для ±2% точности

### 2. ZMPT101B — инерция и фазовый сдвиг

Трансформатор имеет фазовый сдвиг и некоторую инерцию. Для контроля мощности это не критично — ТЭН сам по себе инерционный (секунды на нагрев/остывание).

**Важно:** фильтровать ADC замеры правильно — не использовать слишком большое окно усреднения, иначе реакция на скачок напряжения будет слишком медленной. Оптимально: среднее за 1-2 полупериода (10-20 мс).

### 3. Нелинейная зависимость мощность/угол

Мощность ∝ sin²(θ), где θ — угол открытия. DIMMERLINK принимает линейный уровень 0-10000, но это уровень угла, а не мощности. При уровне 50% мощность будет ~50%, но при уровне 75% мощность будет ~87%.

**Решение:** таблица линеаризации или формула `arcsin(√fraction)` — пересчёт заданной мощности в уровень угла для DIMMERLINK.

### 4. UART конфликт

ESP32 использует UART для лога (Serial0) и может использовать для других задач. DIMMERLINK на отдельном HardwareSerial (Serial1 или Serial2) — конфликта не будет.

### 5. Безопасность

- Гальваническая развязка ZMPT101B — безопасно для ESP32
- DIMMERLINK тоже гальванически развязан (optocoupler)
- В случае зависания ESP32 — DIMMERLINK оставит TRIAC в последнем состоянии (или можно добавить watchdog-таймер на полное отключение)

## Изменения в существующем коде

### Замена текущей логики управления ТЭНом

Сейчас: `outputManager->setPower(true/false)` + 2 пина для выбора режима (вся/предустановленная).

Новая логика: `powerManager->setPower(2000)` — задаёт 2000Вт, PowerManager сам управляет DIMMERLINK и корректирует по ZMPT101B.

### Настройки (SystemConfig)

Новые поля:
- `R_TEN` — сопротивление ТЭНа (Ом), float
- `maxPower` — уже есть (`cfg.n`, 100-3500 Вт)
- `useDimmer` — флаг: true = DIMMERLINK+ZMPT101B, false = старая схема (контактор + 2 пина)

### WEB интерфейс

Вместо двух кнопок "вся мощность / предустановленная":
- **Ползунок 0-100%** или **ввод мощности в Вт**
- Отображение: заданная мощность / расчётная мощность / напряжение сети
- Возможность менять мощность **в процессе** ректификации

### LCD

- Строка с мощностью: "PWR: 2000W / 215V"
- Этап SET_nAS: настройка мощности через кнопки LCD (вместо/дополнительно к внешнему регулятору)

## Этапы внедрения

### Этап 1: Аппаратный
- Покупка DIMMERLINK модуля (25А или 40А)
- Покупка ZMPT101B + делитель + фильтр
- Монтаж на плату, подключение к ESP32
- Тест: управление диммером через UART (без ZMPT101B, фиксированный угол)
- Тест: чтение напряжения с ZMPT101B через ADC

### Этап 2: Код — PowerManager
- Новый класс PowerManager (файлы .h/.cpp)
- Интеграция в BuhloWar.ino (begin, loop)
- Настройка `useDimmer` — переключение между старой и новой схемой
- Тест: задание мощности → проверка что напряжение и мощность корректны

### Этап 3: WEB + LCD
- Ползунок мощности в WEB
- Отображение напряжения/мощности в мониторинге
- LCD: настройка мощности на этапе SET_nAS

### Этап 4: PID (опционально)
- Автоматическое удержание температуры царги за счёт регулировки мощности
- Например: при отбое голов снизить мощность чтобы TSAR не росла
- PID: setpoint = целевая температура, input = TSAR, output = мощность ТЭНа

## Важные замечания

- Текущая схема (контактор + внешний регулятор) **не удаляется** — остаётся как fallback (`useDimmer = false`)
- DIMMERLINK и ZMPT101B — опциональные компоненты, проект работает и без них
- Без ZMPT101B DIMMERLINK тоже usable — просто без стабилизации по напряжению (angle-only, погрешность ±5-7% при колебаниях сети)
- Экономически: оба модуля ~$3-5 суммарно
- Надёжность: гальваническая развязка на обоих модулях, безопасно для ESP32
