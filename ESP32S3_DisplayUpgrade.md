# ESP32-S3 + 7" дисплей (Waveshare) — Анализ для BuhloWar

## Что это

Waveshare выпускает комплекты "ESP32-S3 + 7" дисплей" — это единая плата, где ESP32-S3 подключён к 7" TFT экрану напрямую через параллельный интерфейс (I8080/RGB).

**Характеристики типового комплекта:**
- ESP32-S3, двухъядерный 240 МГц (LX7), 512KB SRAM
- 8MB PSRAM (для framebuffer)
- 7" TFT, 800×480, IPS, ёмкостный тач
- Интерфейс: I8080 параллельный (16-бит, до 40 МГц) или RGB
- MicroSD слот, USB Type-C
- Цена: ~$30-45 на AliExpress

## Ключевое отличие от обычного ESP32

ESP32-S3 имеет **параллельный видеоинтерфейс (I8080)**, которого нет у обычного ESP32. Это позволяет напрямую управлять пикселями 7" экрана без внешнего контроллера.

Для сравнения:
| | ESP32 (текущий) | ESP32-S3 (Waveshare) |
|---|---|---|
| Ядра | Dual 240 МГц Xtensa | Dual 240 МГц LX7 |
| SRAM | 520 KB | 512 KB |
| PSRAM | опционально | 8 MB (встроен) |
| Видеовыход | Нет | I8080 16-бит параллельный до 40 МГц |
| USB | Нет OTG | USB OTG |
| GPIO | 34 | 45 |
| WiFi/BLE | 802.11 b/g/n | 802.11 b/g/n + BLE 5.0 |

## Почему это может работать

**PSRAM 8MB** — главное преимущество. Для 800×480×16бит = 768KB на один framebuffer. С 8MB PSRAM можно хранить несколько буферов и ресурсы UI.

**LVGL (Light and Versatile Graphics Library)** — оптимизирована именно для ESP32-S3. Позволяет создавать полноценный UI с кнопками, графиками, слайдерами, диаграммами. Работает на параллельном интерфейсе с приемлемой скоростью (30-60 FPS для типовых элементов).

**Параллельный интерфейс I8080** — в отличие от SPI (который слишком медленный для 800×480), I8080 даёт пропускную способность ~80 МБ/с — достаточно для плавного UI.

## Проблемы и риски

### 1. Полная замена железа ⚠️ КРИТИЧНО

Это **не модуль**, а **новая плата**. Весь текущий монтаж (реле, датчики, клапаны, buzzer) нужно **переделать** под новую распиновку ESP32-S3. GPIO пины у S3 другие — текущий BuhloWar.ino не запустится без переработки pin map.

**Что нужно переделать:**
- Распиновка всех датчиков DS18B20 (OneWire)
- Распиновка BME280 (I2C)
- Распиновка реле клапанов (OUTPUT)
- Распиновка buzzer
- Распиновка LCD (если оставить как backup) — нет, LCD 2004 подключается по I2C, должно работать
- Кнопки — если оставить
- Питание (ESP32-S3 может отличаться по потреблению)

### 2. Портирование кода ESP32 → ESP32-S3

**Совместимо:**
- Ядро Arduino ESP32 поддерживает S3 (esp32 Arduino core >= 2.0.0)
- WiFi, HTTP сервер — без изменений
- FreeRTOS — без изменений
- I2C, OneWire — без изменений (пины другие, но логика та же)
- EEPROM/Preferences — без изменений

**Требует адаптации:**
- Pin map — все `#define PIN_xxx` нужно переписать
- Библиотеки — большинство совместимы, но некоторые могут потребовать обновления
- Компиляция под ESP32-S3 target в PlatformIO/Arduino IDE

**Примерная оценка портирования:** 1-2 дня на компиляцию и тест пинов, 1-2 дня на проверку всех периферийных устройств.

### 3. LVGL + UI разработка

Нужна полноценная разработка UI на LVGL:
- Экран мониторинга (температуры, давления, графики)
- Экран настроек
- Экран калибровки клапанов
- Экран управления процессом
- Обработчик тач-событий вместо кнопок

**Объём:** ~1000-2000 строк кода для UI + дизайн макетов.

LVGL — мощная библиотека, но кривая обучения крутая. Простой пример:
```cpp
// Температура на экране
lv_label_set_text_fmt(label_tsar, "ЦАРГ: %.1f°C", data.tsar.value);

// График
lv_chart_add_data(chart, lv_chart_add_series(chart, lv_color_hex(0xf59e0b)));

// Кнопка вместо физической
lv_obj_add_event_cb(btn_next, event_handler, LV_EVENT_CLICKED, NULL);
```

### 4. Ресурсы ESP32-S3

LVGL + 800×480 экран — это ощутимая нагрузка:
- LVGL занимает ~100-200KB RAM для объектов UI
- Framebuffer: 2 × 768KB (double buffering) = 1.5MB из 8MB PSRAM
- CPU: отрисовка UI занимает ~10-30% одного ядра
- С учётом двух ядер S3: одно ядро на UI, второе на процесс (loop) — **работоспособно**

Но нужно аккуратно с потоками:
- Core 0: LVGL отрисовка + тач обработка
- Core 1: BuhloWar process loop (sensor reading, valve control, WiFi)

### 5. Раздельный код UI и логики

Сейчас UI (LCD меню) и логика (ProcessEngine) **тесно связаны** — меню напрямую вызывает методы ProcessEngine. С переходом на LVGL нужно разделить:
- **UI слой (LVGL)** — только отображение и сбор ввода
- **Логика (ProcessEngine)** — управление процессом, не знает о UI
- **Bridge** — связывает UI и логику через коллбеки/события

Это хорошая архитектура, но требует **значительного рефакторинга** существующего кода.

## Архитектура если выбирать этот путь

```
ESP32-S3 (Waveshare 7")
├── Core 0 — UI Task (FreeRTOS)
│   ├── LVGL loop (отрисовка 30 FPS)
│   ├── Touch handler
│   ├── Page: Monitor (температуры, давления, графики)
│   ├── Page: Control (управление процессом)
│   ├── Page: Settings (настройки)
│   ├── Page: Calibration (калибровка клапанов)
│   └── Bridge → ProcessEngine
│
├── Core 1 — Process Task (FreeRTOS, текущий loop)
│   ├── ProcessEngine (логика процесса, без изменений)
│   ├── SensorAdapter (DS18B20, BME280)
│   ├── OutputManager (реле, клапаны)
│   ├── AppNetwork (WiFi, WebServer, CloudManager)
│   └── Bridge → UI (обновление данных на экране)
│
├── PSRAM — Framebuffers + UI ресурсы
│   ├── Framebuffer 1 (768KB)
│   ├── Framebuffer 2 (768KB)
│   └── LVGL objects + графики
│
└── Flash — Код + LVGL + шрифты + изображения
```

## Порядок внедрения

### Этап 1: Аппаратный
- Покупка Waveshare ESP32-S3 + 7" комплект
- Перенос монтажа: датчики, реле, клапаны → новые пины S3
- Проверка: все датчики читаются, реле переключаются
- Подключение питания (возможно нужен более мощный БП, 7" дисплей потребляет ~300-500мА)

### Этап 2: Порт кода
- Компиляция BuhloWar под ESP32-S3 target
- Переназначение пинов
- Тест: весь текущий функционал работает на S3 (через Serial Monitor)
- LCD 2004 + кнопки — оставить для отладки

### Этап 3: LVGL интеграция
- Подключение LVGL библиотеки
- Настройка драйвера дисплея (I8080)
- Настройка тач-контроллера
- Простой тест: вывод температуры на экран

### Этап 4: UI разработка
- Проектирование страниц (мониторинг, управление, настройки, калибровка)
- Реализация каждой страницы в LVGL
- Замена обработчиков кнопок на обработчики тач
- Графики (Chart.js аналог на LVGL)

### Этап 5: WEB + 7" дисплей параллельно
- WEB интерфейс (index.html) остаётся — работает через WiFi
- 7" дисплей показывает тот же мониторинг, но локально
- Оба интерфейса обновляются из одних данных

## Итоговая оценка

### Когда это стоит делать:
- Хочешь **единое устройство** — дисплей и контроллер на одной плате
- Планируешь **развитие проекта** (новые экраны, анимации, графики)
- Готов к **значительной переработке** железа и кода
- Хочешь **избавиться от LCD 2004** как основного дисплея

### Когда лучше выбрать другой вариант:
- Нужно быстро получить результат → Android планшет
- Не хочешь переделывать монтаж → Nextion (добавляется к текущему ESP32)
- Текущий ESP32 работает стабильно, не хочется рисковать

### Рекомендация

Если готов инвестировать 3-4 недели — это **самый перспективный путь**. ESP32-S3 + 7" + LVGL даёт полноценный встраиваемый интерфейс. Код будет чище (разделение UI и логики), результат — профессиональный.

**Но не на текущем этапе.** Сначала:
1. Закончить тестирование текущей версии
2. Реализовать PowerManager (DIMMERLINK + ZMPT101B)
3. Затем — переход на ESP32-S3 + 7" как отдельная ветка развития
