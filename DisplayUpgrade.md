# Монитор 7" для BuhloWar — Анализ вариантов замены LCD 2004 + кнопки

## Проблема

Текущее решение: LCD 2004 (20 символов × 4 строки) + 6 физических кнопок. Работает, но мало информации на экране, нет графики, нет каскадных меню, нет диаграмм.

Цель: 7" сенсорный экран для полноценного интерфейса.

## Ключевое ограничение

**ESP32 не имеет видео-выхода** (нет HDMI, LVDS, RGB параллельного порта в достаточном объёме). Поэтому напрямую подключить 7" монитор к ESP32 нельзя. Нужен либо промежуточный контроллер в дисплее, либо другой подход.

## Вариант 1: Nextion NX8048P070 (7" HMI дисплей) ⭐ РЕКОМЕНДУЕМЫЙ

**Что это:** Дисплей со встроенным ARM-контроллером. ESP32 общается с ним по **UART** (отправляет команды). Дисплей сам рисует интерфейс, обрабатывает касания.

**Характеристики:**
- Диагональ: 7", разрешение 800×480, IPS
- Сенсор: ёмкостный (многотач)
- Интерфейс: UART (TX/RX), 5В питание
- Встроенный MCU: ARM Cortex, 16MB Flash для UI
- Цена: ~$35-50 на AliExpress

**Плюсы:**
- ✅ Подключается к ESP32 всего 2 проводами (UART TX/RX) + питание
- ✅ Не грузит ESP32 — дисплей сам рисует графику, обрабатывает тач
- ✅ Редактор Nextion (HMI) для создания UI — визуальный drag-and-drop
- ✅ ESP32 только отправляет: "покажи температуру 78.2°C", "нарисуй график"
- ✅ Возможна анимация, графики, диаграммы — всё делает встроенный MCU
- ✅ Надёжен, проверен в промышленных применениях

**Минусы:**
- ⚠️ Цена выше чем у голого TFT
- ⚠️ Редактор UI закрытый (только Windows, Nextion Editor)
- ⚠️ Нужна переписка всего UI с LCD-меню на Nextion-страницы
- ⚠️ UART ограничивает скорость обновления (~115200 бод)

**Интеграция с BuhloWar:**
- Новый класс `DisplayManager` — абстракция над дисплеем
- Замена всех `lcd->print()` и `menu->display()` на команды Nextion
- Тач-события возвращаются по UART → обработка в ESP32 вместо кнопок
- Объём работы: 300-500 строк кода + создание UI в Nextion Editor
- **LCD 2004 + кнопки остаются как fallback**

**Протокол Nextion (пример):**
```
ESP32 → Nextion:  "t0.txt=\"78.2°C\""     // текст
ESP32 → Nextion:  "j0.val=45"              // прогресс-бар
ESP32 → Nextion:  "add 1,0,0"              // точка на график
Nextion → ESP32:  "0x65 0x01 0x01"        // нажата кнопка page0 id1
```

## Вариант 2: Дешёвый Android планшет 7" + WiFi ⭐ САМЫЙ ПРОСТОЙ

**Что это:** Любой Android планшет 7" ($15-25 на AliExpress). Подключается к WiFi сети ESP32. Открывает WEB интерфейс (index.html) в браузере в kiosk-режиме.

**Плюсы:**
- ✅ **Ноль изменений в коде ESP32** — WEB интерфейс уже готов!
- ✅ Весь интерфейс уже работает: графики, мониторинг, управление
- ✅ Мощный процессор — плавная анимация
- ✅ Ёмкостный тач — лучший из всех вариантов
- ✅ Дёшево
- ✅ Full HD (1280×800) на большинстве планшетов
- ✅ Возможность обновлять UI через веб без прошивки ESP32

**Минусы:**
- ⚠️ Зависит от WiFi — если ESP32 в режиме AP, планшет подключается к ней
- ⚠️ Android может уснуть экран (нужен kiosk-режим / настройка)
- ⚠️ Не выглядит как "встроенный" прибор (видно что это планшет)
- ⚠️ Дополнительное устройство для зарядки

**Интеграция с BuhloWar:**
- Никаких изменений в коде!
- Настройка планшета: установить Fully Kiosk Browser (бесплатно), добавить в автозагрузку, URL = IP ESP32
- LCD 2004 + кнопки остаются как backup если планшет отключили

**Рекомендуемые планшеты:**
- Любой 7" на Allwinner или MediaTek, 2/16GB, ~$15-25
- Важно: IPS экран (не TN), WiFi 2.4GHz
- Пример: Alldocube iPlay 7, Teclast P80, любые no-name

## Вариант 3: 7" TFT + контроллер RA8875 (SPI/параллельный)

**Что это:** Голый 7" TFT-экран + контроллер RA8875. ESP32 рисует каждый пиксель через SPI или параллельный интерфейс.

**Минусы (почему не рекомендуется):**
- ❌ ESP32 будет постоянно рисовать графику — нет ресурсов на основную работу
- ❌ SPI слишком медленный для 800×480 (заметное мерцание при обновлении)
- ❌ Съедает много GPIO пинов (параллельный интерфейс)
- ❌ Нужна сложная библиотека для UI (LVGL, TFT_eSPI)
- ❌ Нет встроенного обработчика тача — отдельный контроллер
- ❌ Объём кода: 500-1000 строк только для отрисовки
- ❌ ESP32 будет тратить 50-80% времени на отрисовку экрана

**Вердикт:** Не подходит для BuhloWar. ESP32 не справится с одновременным управлением процессом и отрисовкой сложного UI на 7" экране.

## Вариант 4: Raspberry Pi + 7" HDMI монитор + ESP32

**Что это:** RPi как медиатор — рендерит UI на HDMI монитор, общается с ESP32 по UART/WiFi/SPI.

**Плюсы:**
- ✅ Полноценный Linux, любой UI фреймворк
- ✅ Отличная графика, видео, веб

**Минусы:**
- ❌ Оверкилл — RPi стоит $35-55 + монитор $25-40 = $60-95
- ❌ Сложная архитектура: два устройства вместо одного
- ❌ RPi требует свой код, свою прошивку, своё обновление
- ❌ Зависимость от SD карты RPi
- ❌ Больше потребление, больше тепло

**Вердикт:** Перегружено. Если нужен большой экран — лучше вариант 2 (планшет).

## Рекомендация

### Путь 1 (быстрый): Android планшет + WiFi
- Покупаешь любой 7" планшет
- Устанавливаешь Fully Kiosk Browser
- Открываешь WEB интерфейс ESP32 в kiosk-режиме
- **Время внедрения: 30 минут**
- **Изменений в коде: 0**

### Путь 2 (профессиональный): Nextion 7"
- Покупаешь Nextion NX8048P070
- Создаёшь UI в Nextion Editor
- Пишешь класс DisplayManager для ESP32
- **Время внедрения: 1-2 недели**
- **Изменений в коде: 300-500 строк + UI дизайн**

### Комбо (лучшее): Оба варианта
- Планшет для повседневного использования (WEB UI уже готов)
- Nextion как встроенный дисплей на корпусе (для автономной работы без WiFi)
- LCD 2004 + кнопки остаются как последний fallback

## Важные замечания

- LCD 2004 + кнопки **не удаляются** из кода — остаются как резервный вариант
- WEB интерфейс (index.html) — не зависит от типа дисплея, работает всегда
- Выбор дисплея не влияет на логику ProcessEngine, SensorAdapter, OutputManager
- Новый класс DisplayManager абстрагирует работу с экраном — легко переключаться между LCD / Nextion / "нет экрана"
