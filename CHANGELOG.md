## Оптимизация CloudManager - ротация запросов 28.03.2026

### Проблема
- loop() блокировался на 0.4-1.7 сек (иногда до 6 сек)
- Причина: CloudManager.update() мог делать до 3 HTTP запросов одновременно
- Каждый запрос имеет таймаут 2 сек
- Худший случай: 3 запроса × 2 сек = 6 сек блокировки

### Решение
- Переделан метод update() на ротацию запросов
- Теперь только ОДИН запрос за вызов
- Максимальная блокировка: 2 сек (вместо 6 сек)

### Изменения в CloudManager.h

1. **Добавлена переменная ротации**
   ```cpp
   int nextRequestType = 0;  // 0=telemetry, 1=commands, 2=settings
   ```

2. **Переделан update()**
   - Было: все 3 запроса могли сработать одновременно
   - Стало: switch-case с ротацией, только один запрос за вызов

3. **Увеличен интервал settings**
   - Было: 30 сек
   - Стало: 60 сек (снижение нагрузки)

4. **Исправлены checkCommands() и checkSettings()**
   - Теперь возвращают true при любом HTTP 200 (не только при наличии данных)
   - Таймеры обновляются корректно

### Влияние на систему

| Компонент | Было | Стало |
|-----------|------|-------|
| Макс. блокировка loop | 6 сек | 2 сек |
| Обновление телеметрии | 2 сек | ~4 сек |
| Проверка команд | 2 сек | ~4 сек |
| Локальный Web (SD) | 1 сек | 1 сек (без изменений) |

- CloudManager.h: ротация запросов, интервал settings 60 сек
